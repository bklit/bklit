Get Bklit Analytics running locally in minutes. This guide will help you set up the dashboard, SDK, and playground for development.

<Callout type="info">
  **Quick Setup:** To get started quickly, you only need the SDK, dashboard, and
  playground running. The website and Trigger.dev are optional for basic
  development.
</Callout>

<Callout type="success">
  **Local Development:** For isolated local databases with Docker and Prisma Dev, see [Local Development](/reference/local-development).
</Callout>

## Prerequisites

- Node.js 22.14.0 or higher
- pnpm 9.6.0 or higher (recommended)
- PostgreSQL (<a href="https://console.prisma.io/sign-up" target="_blank">Prisma Postgres</a> recommended)
- GitHub OAuth app (for authentication)

## Environment Setup

### Local Development (Recommended for Start)

For local development with isolated databases, use DEV_* prefixed variables. Create a `.env` file in the root directory:

```bash
NODE_ENV="development"

# Local Databases
DEV_DATABASE_URL="prisma+postgres://localhost:51213/?api_key=__API_KEY__"
DEV_CLICKHOUSE_HOST="http://localhost:8123"
DEV_CLICKHOUSE_USERNAME="default"
DEV_CLICKHOUSE_PASSWORD="local_dev_password"
DEV_BKLIT_DEFAULT_PROJECT="your-local-demo-project-id"

# Production Databases (for deployment later)
DATABASE_URL="postgresql://..."
CLICKHOUSE_HOST="https://..."
CLICKHOUSE_USERNAME="default"
CLICKHOUSE_PASSWORD="..."
BKLIT_DEFAULT_PROJECT="your-prod-project-id"

# Auth (works for both local and prod)
AUTH_SECRET="your-secret"
AUTH_GITHUB_ID="your-github-id"
AUTH_GITHUB_SECRET="your-github-secret"

# Polar - use sandbox for local dev
POLAR_SERVER_MODE="sandbox"
POLAR_ACCESS_TOKEN="..."
POLAR_WEBHOOK_SECRET="..."

# Other required variables
ALERT_EMAIL="your-email@example.com"
HEALTH_CHECK_SECRET="your-secret"
NEXT_PUBLIC_MAPBOX_TOKEN="your-token"
```

See [Local Development](/reference/local-development) for complete setup guide.

## Database Setup

### Local Development with Prisma Dev

1. **Start Prisma Postgres locally:**
   ```bash
   cd apps/dashboard
   npx prisma dev
   ```
   
2. **Press `h`** to get the connection URL

3. **Add to `.env`** as `DEV_DATABASE_URL`

4. **Start ClickHouse** (automatically via docker-compose):
   ```bash
   pnpm dev:services
   ```

5. **Run migrations:**
   ```bash
   pnpm db:migrate
   cd packages/analytics && pnpm run migrate
   ```

See [Local Development](/reference/local-development) for detailed instructions.

### Production Setup

For production deployment with Prisma Postgres or your PostgreSQL provider:

```bash
pnpm db:generate
pnpm db:migrate
```
